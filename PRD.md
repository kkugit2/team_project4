# 채용 정보 플랫폼 PRD (Product Requirements Document)

## 1. 프로젝트 개요

**채용 정보 플랫폼**은 개인의 학적, 경력, 자격증 정보를 기반으로 채용공고를 추천하고, 원티드 OpenAPI의 AI 예측 기능을 활용하여 합격 확률을 실시간으로 제시하는 웹 기반 채용 정보 플랫폼입니다. 사용자는 여러 기업의 채용공고를 대시보드 형식으로 한눈에 비교하고, 자신의 경력 정보를 관리하며, 관심 채용공고를 찜하고 지원현황을 추적할 수 있습니다. 이를 통해 구직자는 자신에게 최적화된 일자리를 빠르게 찾고, 지원 전략을 수립할 수 있습니다.

---

## 2. 타깃 사용자 분석

### 페르소나 1: 신입/경력직 구직자 (주요 사용층)
- **특징**: 대학교 졸업자, 전직 또는 첫 직장을 찾고 있는 사람 (22~35세)
- **주요 니즈**:
  - 자신의 학력과 경력으로 합격할 수 있는 기업이 어디인지 빠르게 파악하고 싶음
  - 여러 기업의 채용공고를 한곳에서 비교하고 싶음
  - 지원현황을 체계적으로 관리하고 싶음
  - 특정 공고에 대한 합격 확률을 미리 예측받고 싶음

### 페르소나 2: 직무 적성 탐색 중인 현직자
- **특징**: 이직을 고려하거나 경력 전환을 생각 중인 직장인 (25~40세)
- **주요 니즈**:
  - 자신의 경력이 어떤 직무에 적합한지 알고 싶음
  - 새로운 기업이나 산업의 채용정보를 효율적으로 탐색하고 싶음
  - 자신의 경력 및 자격증 정보를 최신으로 유지하고 싶음

## 3. 핵심 기능 목록

| 기능 | 설명 | 우선순위 |
|------|------|--------|
| **로그인/회원가입** | Supabase 인증을 활용한 로그인/회원가입 기능. 회원가입 시 아이디, 비밀번호, 학적, 학점, 자격증, 경력 정보 입력 (모든 항목 필수이지만 '해당 사항 없음' 선택 가능) | P0 (필수) |
| **랜딩페이지/대시보드** | 여러 기업의 채용공고를 카드 형식의 대시보드로 표시. 각 카드에는 기업명, 채용 포지션, 합격 예측 확률, 찜 상태, 지원 현황이 표시됨 | P0 (필수) |
| **합격 확률 예측** | 원티드 OpenAPI의 AI 예측 API('서류합격예측')를 활용하여 각 공고마다 사용자의 합격 확률을 실시간 계산 및 표시 | P0 (필수) |
| **기업 상세 정보 조회** | 대시보드의 기업 카드를 클릭 시 해당 기업의 상세 소개 페이지로 이동. 기업 정보, 채용 포지션 목록을 원티드 API('기업 상세 조회', '기업별 채용 포지션 목록')로부터 조회 | P0 (필수) |
| **지원하기 연결** | 기업 상세 페이지의 '지원하기' 버튼 클릭 시 해당 기업의 실제 지원서 작성 사이트로 외부 링크 이동 | P0 (필수) |
| **채용공고 찜하기** | 사용자가 관심 있는 채용공고를 찜하고, 찜한 목록을 별도로 조회 및 관리 (DB에 저장) | P1 (높음) |
| **지원현황 추적** | 사용자가 지원한 기업 및 채용공고의 현황을 시각적으로 표시 (지원 완료, 진행 중 등) | P1 (높음) |
| **마이페이지** | 사용자의 학적, 학점, 자격증, 경력 정보를 조회, 수정, 삭제 가능. 프로필 정보 변경 시 합격 확률 재산정 가능 | P1 (높음) |
| **포지션 탐색/필터링** | 원티드 API('포지션 리스트 V2')를 활용하여 직군, 직무, 스킬 등으로 채용공고 필터링 | P2 (중간) |
| **스킬/직군 마스터 데이터** | 원티드 API('태그 마스터: 직군/직무, 스킬 리스트')를 활용하여 필터링 및 매칭에 사용되는 공식 데이터 반영 | P2 (중간) |

## 4. 사용자 플로우

### 플로우 1: 초기 사용자 (회원가입 및 첫 채용공고 탐색)

1. **회원가입 페이지 진입**
   - 랜딩페이지 또는 로그인 페이지에서 '회원가입' 버튼 클릭
   - 회원가입 폼 표시 (아이디, 비밀번호, 학적, 학점, 자격증, 경력 정보)
   - 모든 항목은 필수이지만 '해당 사항 없음' 선택 가능

2. **정보 입력 및 계정 생성**
   - 사용자가 필요한 정보 입력
   - Supabase 인증 API로 계정 생성
   - 회원가입 완료 → 자동으로 로그인 또는 로그인 페이지로 리다이렉트

3. **로그인**
   - 아이디(이메일 또는 사용자명)와 비밀번호로 로그인
   - 로그인 성공 시 랜딩페이지(대시보드)로 이동

4. **랜딩페이지(대시보드) 진입**
   - 여러 기업의 채용공고가 카드 형식으로 표시
   - 각 카드에는 다음 정보 포함:
     - 기업명, 채용 포지션명
     - 합격 예측 확률 (원티드 API 활용)
     - 찜 아이콘 (클릭 시 찜 추가/제거)
     - 지원 현황 표시 (예: "지원 완료", "미지원")

5. **채용공고 탐색 (선택사항)**
   - 필터 기능을 사용하여 직군, 직무, 스킬 등으로 공고 필터링
   - 원티드 API의 '포지션 리스트 V2' 활용

6. **기업 상세 정보 조회**
   - 대시보드의 기업 카드 또는 공고명 클릭
   - 기업 상세 페이지로 이동 (기업 소개, 근무지, 복리후생 등)
   - 해당 기업의 채용 포지션 목록 표시

7. **찜하기**
   - 기업 상세 페이지 또는 대시보드에서 하트 아이콘 클릭
   - 채용공고 찜 → DB에 저장
   - 찜 아이콘이 활성화 상태로 변경

8. **지원하기**
   - 기업 상세 페이지의 '지원하기' 버튼 클릭
   - 해당 기업의 실제 지원서 작성 사이트로 외부 링크 이동 (새 창)
   - 지원 완료 후 돌아와서 지원 현황 업데이트 (선택적으로 사용자가 수동 업데이트 또는 자동 추적)

---

### 플로우 2: 마이페이지 - 프로필 관리

1. **마이페이지 진입**
   - 헤더 또는 메뉴의 '마이페이지' 링크 클릭
   - 사용자의 현재 프로필 정보 조회 (학적, 학점, 자격증, 경력)

2. **정보 수정**
   - 각 항목의 '수정' 버튼 클릭
   - 수정 폼 팝업 또는 인라인 에디트 표시
   - 변경 사항 저장 → DB 업데이트
   - 프로필 정보 변경 시 기존 합격 확률 재산정 (선택사항)

3. **정보 삭제**
   - 특정 항목(예: 자격증, 경력 경력)의 '삭제' 버튼 클릭
   - 삭제 확인 팝업 표시
   - 확인 후 삭제 → DB에서 제거

4. **마이페이지 종료**
   - 뒤로가기 또는 메뉴에서 다른 페이지로 이동

---

### 플로우 3: 찜한 공고 조회

1. **찜한 공고 목록 진입**
   - 대시보드 또는 메뉴에서 '찜한 공고' 또는 '북마크' 링크 클릭
   - 사용자가 찜한 모든 채용공고를 리스트 형식으로 표시

2. **찜한 공고 관리**
   - 각 공고의 찜 제거 버튼 클릭 시 목록에서 삭제
   - 공고명 클릭 시 기업 상세 페이지로 이동

## 5. 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| **프론트엔드** | HTML/CSS/JavaScript (바닐라) | 프로젝트 요구사항에 맞춘 가벼운 구현. 복잡한 상태 관리가 필요 없는 경우 바닐라 JS로 빠른 개발 가능. 필요시 나중에 React/Vue 등으로 마이그레이션 가능 |
| **백엔드** | Node.js + Express | 원티드 API 호출 및 인증 로직 구현. JavaScript 통일로 개발 효율성 향상 |
| **데이터베이스** | PostgreSQL (Supabase) | 관계형 데이터 구조에 최적화. Supabase의 내장 인증, 실시간 기능 활용 가능 |
| **인증** | Supabase Auth | 회원가입/로그인 기능 안전하게 구현. OAuth 지원 및 세션 관리 용이 |
| **외부 API** | 원티드 OpenAPI (V1 + V2) | AI 예측, 기업 정보, 포지션 탐색, 스킬/직군 마스터 데이터 활용 |
| **호스팅** | 가정: Vercel (프론트엔드) / Heroku 또는 AWS (백엔드) | 별도 명시 없어 일반적인 구성으로 가정 |

**기술 스택 선택 가정**: 백엔드와 호스팅 방식에 대해 요구사항이 명시되지 않아, Node.js/Express 스택과 일반적인 클라우드 호스팅을 가정하고 작성하였습니다.

### 6.2 테이블 상세 정의

#### 1. **User** (사용자)
| 컬럼명 | 타입 | 설명 | 제약 |
|--------|------|------|-----|
| id | UUID | 사용자 고유 ID | PK, Supabase 자동 생성 |
| email | VARCHAR(255) | 사용자 이메일/로그인 아이디 | UNIQUE, NOT NULL |
| username | VARCHAR(100) | 사용자명 | NOT NULL |
| password_hash | VARCHAR(255) | 비밀번호 해시 | NOT NULL, Supabase 관리 |
| created_at | TIMESTAMP | 계정 생성 시간 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 마지막 수정 시간 | DEFAULT CURRENT_TIMESTAMP |

#### 2. **UserProfile** (사용자 프로필)
| 컬럼명 | 타입 | 설명 | 제약 |
|--------|------|------|-----|
| id | UUID | 프로필 고유 ID | PK |
| user_id | UUID | 사용자 ID | FK (User.id), NOT NULL |
| education | VARCHAR(255) | 학적 (예: 서울대학교 컴퓨터공학과) | NULL 가능 ('해당 사항 없음' 선택 시) |
| gpa | DECIMAL(3,2) | 학점 (0.00 ~ 4.50) | NULL 가능 |
| created_at | TIMESTAMP | 생성 시간 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 마지막 수정 시간 | DEFAULT CURRENT_TIMESTAMP |

#### 3. **Certification** (자격증)
| 컬럼명 | 타입 | 설명 | 제약 |
|--------|------|------|-----|
| id | UUID | 자격증 고유 ID | PK |
| user_id | UUID | 사용자 ID | FK (User.id), NOT NULL |
| cert_name | VARCHAR(255) | 자격증명 (예: TOEIC, 정보처리기사) | NOT NULL |
| issued_date | DATE | 취득 일자 | NOT NULL |
| expires_at | DATE | 만료 일자 | NULL 가능 (만료 없음) |
| created_at | TIMESTAMP | 생성 시간 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 마지막 수정 시간 | DEFAULT CURRENT_TIMESTAMP |

#### 4. **Experience** (경력)
| 컬럼명 | 타입 | 설명 | 제약 |
|--------|------|------|-----|
| id | UUID | 경력 고유 ID | PK |
| user_id | UUID | 사용자 ID | FK (User.id), NOT NULL |
| company_name | VARCHAR(255) | 회사명 | NOT NULL |
| job_title | VARCHAR(255) | 직책/직무 | NOT NULL |
| start_date | DATE | 근무 시작일 | NOT NULL |
| end_date | DATE | 근무 종료일 | NULL 가능 (현직) |
| description | TEXT | 업무 설명 | NULL 가능 |
| created_at | TIMESTAMP | 생성 시간 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 마지막 수정 시간 | DEFAULT CURRENT_TIMESTAMP |

#### 5. **SavedJob** (찜한 채용공고)
| 컬럼명 | 타입 | 설명 | 제약 |
|--------|------|------|-----|
| id | UUID | 찜 기록 고유 ID | PK |
| user_id | UUID | 사용자 ID | FK (User.id), NOT NULL |
| job_posting_id | VARCHAR(255) | 원티드 API의 채용공고 ID | NOT NULL |
| company_name | VARCHAR(255) | 기업명 | NOT NULL |
| position_name | VARCHAR(255) | 채용 포지션명 | NOT NULL |
| created_at | TIMESTAMP | 찜 시간 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 마지막 업데이트 시간 | DEFAULT CURRENT_TIMESTAMP |

### 6.3 관계 설명
- **1:N 관계**:
  - User ↔ UserProfile: 1명의 사용자는 1개의 프로필 정보 보유
  - User ↔ Certification: 1명의 사용자는 여러 개의 자격증 보유 가능
  - User ↔ Experience: 1명의 사용자는 여러 개의 경력 기록 보유 가능
  - User ↔ SavedJob: 1명의 사용자는 여러 개의 채용공고 찜 가능

## 7. 개발 우선순위 및 단계별 계획

### 🎯 MVP (Minimum Viable Product) - Phase 1
**목표**: 사용자가 채용공고를 탐색하고 합격 확률을 확인할 수 있는 기본 서비스 제공

**포함 기능**:
1. 로그인/회원가입 (Supabase 활용)
   - 회원가입 시 필수 정보 입력 (학적, 학점, 자격증, 경력)
   - Supabase 인증 연동

2. 랜딩페이지/대시보드
   - 채용공고 대시보드 UI 구현
   - 원티드 API 연동 (기업 정보, 포지션 리스트)

3. 합격 확률 예측
   - 원티드 OpenAPI의 'AI 예측 - 서류합격예측' 연동
   - 각 공고마다 합격 확률 실시간 계산

4. 기업 상세 정보 페이지

## 8. 유의사항 및 제약조건

### API 사용 유의사항
- **원티드 OpenAPI 인증**: API 키 관리 및 Rate Limiting 고려 필요
- **API 비용**: 원티드 API 호출량에 따른 비용 발생 가능 (별도 협의 필요)
- **데이터 동기화**: 원티드 API의 채용공고 데이터는 주기적으로 동기화하거나, 실시간 조회 구현 필요

### 보안 고려사항
- **비밀번호 관리**: Supabase의 내장 보안 기능 활용 (해싱, 암호화)
- **HTTPS**: 모든 통신은 HTTPS로 암호화 필요
- **사용자 데이터**: GDPR, CCPA 등 개인정보보호 규정 준수

### 성능 고려사항
- **캐싱**: 원티드 API 응답 캐싱으로 응답 속도 개선
- **이미지 최적화**: 기업 로고, 배너 이미지 압축 및 CDN 활용
- **데이터베이스 인덱싱**: 자주 조회하는 컬럼에 인덱스 추가

### 향후 확장 계획
- 모바일 앱 개발 (React Native 등)
- 고급 필터링 및 정렬 옵션 추가
- 채용 통계 및 시장 분석 기능
- 사용자 커뮤니티 기능 (면접 후기, 연봉 정보 등)

## 부록: 가정사항 정리

| 항목 | 가정사항 |
|------|--------|
| **백엔드 프레임워크** | Node.js + Express (명시되지 않아 일반적 선택지로 가정) |
| **호스팅 환경** | Vercel (프론트엔드) / Heroku 또는 AWS (백엔드) (명시되지 않아 일반적 선택지로 가정) |

| **회원가입 정보 저장** | 모든 정보(학적, 학점, 자격증, 경력)는 Supabase PostgreSQL DB에 저장 |

| **지원현황 추적 방식** | 사용자가 지원하기 완료 후 수동으로 상태 업데이트하거나, 향후 자동 추적 기능 추가 고려 |
