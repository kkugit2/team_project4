<개요>
채용 정보 플랫폼

<기능 - 구직자>
1. 랜딩페이지: 여러 기업들이 대시보드 형태로 제시됨 --> 합격 확률 한 번에 확인 가능(원티드 API 이용) + 지원현황, 찜 기능 추가
2. 대시보드를 클릭하면 회사 소개를 확인할 수 있음. 또한 실제 '지원하기' 버튼을 클릭하면 실제로 해당 기업의 지원서 작성 사이트로 이동하게 됨.
3. 마이페이지: 나의 경력 및 자격증을 확인할 수 있음. 회원가입 시 입력한 정보를 기반으로 함.
4. 로그인/회원가입 페이지(Supabase 이용) --> 회원가입 페이지에 개인정보를 입력해야 함(아이디, 비밀번호, 학적, 학점, 자격증, 경력)
5. 지원서 작성 후 분석: 자소서 입력 시 평가(피드백, 나의 장단점, 경쟁자 비교분석 제공) --> 기업은 후술할 서비스를 제공받기 위해 당사에 원하는 인재상 데이터를 제공함. 이를 바탕으로 피드백 제공

<기능 - 기업 인사팀>
1. 로그인/회원가입 페이지: 회원가입 시, 기업인인지 구직자인지 선택해서 할 수 있음. 원래는 별도의 인증절차를 거쳐야하지만 MVP인 만큼 그 부분은 생략. 회원가입의 경우, 회사가 원하는 인재상을 저장해둘 수 있음(ex. 학점, 기술스택, 유사 경험, 인턴십 경험 유무 등)
2. 랜딩페이지: 구직자들은 상술한 서비스를 제공받기 위해 당사에 자기소개서 정보를 제공함. 기업에서는 이 자기소개서를 대시보드 형식으로 볼 수 있음. 대시보드에는 기업이 회원가입 시 입력해둔 인재상과 어느정도 일치하는지 확인할 수 있음. 또한 해당 구직자가 우리 회사에 지원하였는지 여부를 확인할 수 있음. 
3. **다시 생각해보니 기업측에서 어떻게 이득을 얻을 수 있는 구조인지 잘 모르겠습니다. 물론 인재를 확인할 수는 있지만, 그렇다고 해서 스카웃을 할 수 있는 것도 아니고 면접을 프리패스시킬 수 있는 것도 아닙니다. 그게 실제로 가능한 건진 잘 모르겠다만 아무튼 조금 더 고민해 볼 부분인 것 같습니다.
4. **번외로 일단 이 기능들을 모두 구현한 이후에 초기 이용자들을 어떻게 모을 것인지 전략을 세울 필요가 있어보입니다. 

<업무 분담>
1. 공통: 계획서를 바탕으로 각자 PRD.md(전체적 가치와 방향성)를 제작하여 하나로 통합한다. 또한 디자인 방향성을 확장지어 UI-UX-Guideline.md(프론트엔드 개발) 파일 하나로 통합한다. 또한 Backend-Guideline.md(백엔드 개발) 파일에 백엔드 관련 정보를 정리하여 저장한다. 바이브코딩으로 코딩을 할 때에는 CLAUDE.md 파일을 참고하되, 이곳에는 프론트엔드 파일의 위치와 백엔드 파일의 위치를 기록해두고 전체적인 코딩의 구조를 적어두도록 한다. 
ex) CLAUDE.md

[준수 사항]

- 코드를 모두 작성한 뒤에는 반드시 테스트를 진행하며, 프론트엔드와 백엔드 각각, 그리고 프론트엔드와 백엔드를 함께 진행하며, 문제 발생 시 반드시 보고할 것
- 코드 작성 서브에이전트의 작업이 끝나고 나면 작업 종료를 보고하고, 곧바로 깃허브 push 서브에이전트가 작업을 이어갈 것
