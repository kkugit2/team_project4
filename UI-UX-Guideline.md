**대상**: 프론트엔드 개발자

## 1. 디자인 철학 및 컨셉

### 1.1 핵심 가치

**"신뢰 가능한 경력 플랫폼: 명확하고 효율적인 정보 구조"**

경력을 다루는 B2B 채용 플랫폼으로서 **신뢰성**과 **명확성**을 최우선으로 한다. 과도한 시각 효과나 유행 트렌드보다는 정보 계층 구조를 명확히 하고 사용자의 의도를 빠르게 달성할 수 있도록 설계한다.

### 1.2 설계 원칙

- **명확성 우선**: 화면을 스캔할 때 1~2초 내에 정보 계층을 파악 가능
- **최소 인지 부하**: 불필요한 장식 최소화, 기능적 요소만 강조
- **접근성 기반**: WCAG 2.1 AA 수준 이상, 명도 대비 4.5:1 이상 필수
- **효율성**: 각 페이지는 주요 목표를 3~4 단계 내에 완료
- **일관성**: 모든 페이지에서 동일한 컴포넌트, 색상, 간격 시스템 적용

### 1.3 참고 레퍼런스

- **로켓펀치**: 네트워크 기반 채용 플랫폼의 질서 있는 정보 구조
- **원티드**: IT 직군 특화 플랫폼의 명확한 카드 레이아웃
- **2026 디자인 트렌드**: 접근성/윤리적 디자인, 벤토 그리드 레이아웃 참고 (과도한 장식 효과 제외)

### 2.4 컬러 사용 규칙

- **명도 대비**: 모든 텍스트-배경 조합은 최소 4.5:1 이상 (WCAG AA)
- **색상 중복 방지**: 색상만으로 정보 전달 금지, 아이콘/텍스트 라벨 함께 사용
- **색상 보충**: Primary Blue와 Green의 명도 차이 20% 이상 (색맹 고려)
- **배경 기준**: 흰색(#FFFFFF) 또는 Light Gray(#F5F7FA), 텍스트는 항상 Dark Gray(#2C3E50) 이상 사용

## 4. 레이아웃 원칙

### 4.1 그리드 시스템

- **기본 단위**: 8px 기본 그리드
- **컨테이너 너비**:
  - 모바일 (320-479px): 100% - 32px 패딩
  - 태블릿 (480-1023px): 100% - 48px 패딩
  - 데스크톱 (1024px+): 최대 1280px, 중앙 정렬

### 4.2 여백 시스템 (8px 배수)

| 단위 | px | 용도 |
|------|-----|------|
| **xs** | 4px | 인라인 요소 간 간격 |
| **sm** | 8px | 버튼 내부 패딩, 요소 간 타이트한 간격 |
| **md** | 16px | 카드 패딩, 섹션 내 간격 |
| **lg** | 24px | 섹션 간 간격 |
| **xl** | 32px | 주요 섹션 분리 |
| **2xl** | 48px | 페이지 상하 여백, 헤더 높이 |

### 4.3 반응형 설계

**모바일 (320px - 479px)**
- 단일 컬럼 레이아웃
- 버튼 높이: 48px (터치 타겟)
- 측면 패딩: 16px
- 이미지 너비: 100%

**태블릿 (480px - 1023px)**
- 2컬럼 그리드 가능
- 헤더 네비게이션 표시
- 측면 패딩: 24px

**데스크톱 (1024px+)**
- 3컬럼 이상 그리드
- 풀 헤더 네비게이션
- 컨테이너 최대 너비: 1280px
- 측면 패딩: 40px

### 4.4 헤더/네비게이션

- **높이**: 60px (스티키)
- **배경**: White (#FFFFFF), 하단 보더 1px Light Gray
- **구성**: 로고 (좌측) - 메뉴 (중앙) - 사용자 메뉴 (우측)

### 6.3 주요 상호작용

#### 사용자 타입 선택 카드

- 기본: 보더 1px Medium Gray, 배경 White
- 호버: 보더 2px Primary Blue, 배경 Light Gray
- 선택: 보더 2px Primary Blue, 배경 Light Gray, 좌측 4px 파란색 바
- 비선택: 보더 1px Medium Gray, 배경 White, 텍스트 Medium Gray

#### 비밀번호 유효성 표시

```
입력 중 실시간 업데이트 (debounce 300ms):

미충족: 
  아이콘: ○ (회색)
  텍스트: "8자 이상" (회색)

충족:
  아이콘: ✓ (초록색)
  텍스트: "8자 이상" (어두운 회색)
```

#### 진행도 표시

- 진행률 막대: 배경 Light Gray, 채워진 부분 Primary Blue
- 높이: 4px
- 텍스트: "1/3", "2/3", "3/3" (Body Small, 우측 정렬)

#### 보기 아이콘 토글

- 기본: 눈 아이콘 (outline)
- 호버: 배경 Light Gray (10% opacity)
- 클릭: 눈 아이콘 + 슬래시 (비밀번호 보임)
- 색: Primary Blue

### 6.4 입력 필드 검증

| 필드 | 검증 방식 | 오류 메시지 |
|------|---------|-----------|
| **이메일** | 형식 검증 + 중복 확인 (API) | "유효한 이메일을 입력하세요" / "이미 가입된 이메일입니다" |
| **비밀번호** | 실시간 규칙 확인 | "8자 이상, 특수문자 포함 필요" |
| **비밀번호 확인** | 일치 여부 | "비밀번호가 일치하지 않습니다" |
| **필수 필드** | 공백 검증 | "[필드명]은 필수 항목입니다" |

### 6.5 접근성 고려사항

- **키보드 네비게이션**: Tab으로 모든 입력필드 및 버튼 순회 가능, Shift+Tab으로 역순 이동
- **포커스 표시**: 2px solid Primary Blue + 2px offset
- **라벨 연결**: 모든 입력필드는 `<label for="id">` 또는 `aria-label` 포함
- **오류 알림**: `aria-live="polite"` + `role="alert"` (스크린 리더 읽음)
- **명도 대비**: 모든 텍스트 4.5:1 이상 (WCAG AA)
- **터치 타겟**: 모든 버튼 최소 44px × 44px

```
  "로그인하면 당신의 적합도를 확인할 수 있습니다."
  [로그인 하기]  Primary (48px)

지원 버튼:
  [로그인하여 지원하기]  Primary (48px)
```

#### 로그인 + 지원 완료 상태

```
지원 완료 버튼:
  배경: Primary Green (#2ECC40)
  텍스트: White, "지원됨"
  아이콘: 체크마크
  disabled: true (클릭 불가)
```

#### 로딩 상태

```
기업 정보:
  - 스켈레톤 로더 (회색 플레이스홀더)
  - 깜박임 애니메이션 (600ms)

포지션 리스트:
  - 3개의 스켈레톤 카드
```

### 8.5 외부 링크 처리

```
지원하기:
  - 링크: 원티드 공고 URL
  - 새 창: target="_blank" rel="noopener noreferrer"
  - 아이콘: 외부 링크 아이콘 (→)

공고 보기:
  - 링크: 포지션 상세 페이지
  - 새 창: target="_blank"
  - 아이콘: 외부 링크 아이콘

회사 홈페이지:
  - 링크: 기업 공식 웹사이트
  - 새 창: target="_blank"
  - aria-label: "새 창에서 열기"
```

### 8.6 접근성 고려사항

- **포커스**: 모든 버튼 및 링크에 포커스 표시 (2px 파란 테두리)
- **라벨**: 새 창 열기 버튼 `aria-label="새 창에서 열기"`
- **색상**: 합격 확률은 색상 + 텍스트 라벨 ("75% - 높음")
- **명도 대비**: 배경 이미지 위 텍스트는 어두운 오버레이 필수
- **키보드**: Tab, Enter, Shift+Enter (새 창) 지원
- **이미지**: 로고 및 배경 이미지는 `alt` 텍스트 포함

## 9. 인터랙션 및 애니메이션

### 9.1 기본 원칙

- **성능 우선**: 60fps 유지, GPU 가속 활용 (transform, opacity)
- **의도 명확**: 애니메이션은 상태 변화 또는 피드백 전달 목적만
- **최소 지속시간**: 200ms 이상 (빠른 반응), 500ms 이하 (대기 최소)
- **Easing**: `ease`, `ease-in-out`, `ease-out` 사용 (linear 최소화)

### 9.2 주요 애니메이션

| 요소 | 상태 변화 | 지속 | Easing | 설명 |
|------|---------|------|--------|------|
| **버튼** | Hover | 200ms | ease-out | 색상 변경 + 1px 상승 |
| **입력필드** | Focus | 200ms | ease-out | 보더 색상 + 박스 섀도우 |
| **카드** | Hover | 200ms | ease-out | 보더 색상 + 섀도우 + translateY(-2px) |
| **모달** | 등장 | 200ms | ease-out | opacity 0→1 + scale 0.95→1 |
| **모달** | 퇴출 | 150ms | ease-in | opacity 1→0 + scale 1→0.95 |
| **토스트** | 등장 | 300ms | ease-out | translateX 400px→0 + opacity |
| **찜 아이콘** | 토글 | 300ms | ease-in-out | 하트 scale + fill animation |
| **로딩 스피너** | 회전 | 1000ms | linear | 무한 회전 |
| **스켈레톤** | 깜박임 | 600ms | ease-in-out | opacity 0.6~1 |

### 9.3 페이지 전환

```
페이지 진입:
  - 콘텐츠: opacity 0→1 (200ms ease)
  - 스크롤: top으로 자동 이동

페이지 퇴출:
  - opacity 1→0 (100ms ease)
  - 새 페이지 로드 후 진입 애니메이션

모달:
  - 진입: opacity 0→1 + scale 0.95→1 (200ms ease-out)
  - 퇴출: opacity 1→0 + scale 1→0.95 (150ms ease-in)

토스트:
  - 진입: translateX 400px→0 + opacity 0→1 (300ms ease-out)
  - 퇴출: translateX 0→400px + opacity 1→0 (150ms ease-in)
```

## 10. 접근성 (WCAG 2.1 AA)

### 10.1 명도 대비

**최소 요구사항**:
- **일반 텍스트** (14px 미만): 4.5:1
- **큰 텍스트** (18px 이상 또는 14px Bold 이상): 3:1
- **그래픽 요소**: 3:1

### 10.2 키보드 네비게이션

- **Tab**: 순서대로 포커스 이동
- **Shift+Tab**: 역순 포커스 이동
- **Enter**: 버튼/링크 활성화
- **Escape**: 모달/드롭다운 닫기
- **Space**: 체크박스/라디오 토글
- **화살표 키**: 드롭다운 옵션 선택

**포커스 표시**:
```css
outline: 2px solid #0052CC;
outline-offset: 2px;
```

### 10.3 스크린 리더 지원 (ARIA)

```html
<!-- 텍스트 없는 버튼 -->
<button aria-label="다이얼로그 닫기">×</button>

<!-- 에러 메시지 (자동 읽음) -->
<div role="alert" aria-live="polite">
  이메일 형식이 올바르지 않습니다.
</div>

<!-- 헬퍼 텍스트 -->
<input aria-describedby="pwd-hint" />
<small id="pwd-hint">8자 이상, 대문자 포함</small>

<!-- 진행률 -->
<div role="progressbar" 
     aria-valuenow="75" 
     aria-valuemin="0" 
     aria-valuemax="100"
     aria-label="합격 확률"></div>

<!-- 외부 링크 -->
<a href="..." target="_blank" rel="noopener noreferrer"
   aria-label="새 창에서 열기">공고 보기</a>
```

### 10.4 폼 접근성

- **라벨**: 모든 입력필드는 `<label>` 또는 `aria-label` 필수
- **필수 표시**: 시각적(*) + 텍스트("필수")
- **에러 메시지**: `aria-live="polite"`, `role="alert"` 사용
- **입력 유효성**: 실시간 feedback (색상 + 텍스트 + 아이콘)

### 10.5 시각적 접근성

- **색상만 의존 금지**: 색상 + 텍스트 + 아이콘 조합
- **텍스트 크기**: 최소 14px (본문), 12px (메타)
- **줄 높이**: 1.4 이상 (가독성)
- **자간**: 약간의 여유 (약 0.5px)

### 10.6 모바일 접근성

- **터치 타겟**: 최소 44px × 44px
- **줌 가능**: `<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=yes">`
- **텍스트 선택**: `user-select: auto` (기본값 유지)
