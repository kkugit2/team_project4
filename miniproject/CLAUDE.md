# CLAUDE.md

바이브코딩(Claude Code 등)으로 이 프로젝트를 작업할 때 반드시 참고할 최상위 지침 파일입니다.
모든 서브에이전트는 작업 시작 전 이 문서를 먼저 읽습니다.

## 프로젝트 개요
원티드 API 기반 채용 매칭 플랫폼(MVP). 구직자-기업 양방향 매칭, 스카웃 기능 포함.
상세 요구사항은 `PRD.md` 참고.

## 기술 스택
- Frontend: HTML, CSS, JavaScript
- Backend / DB / Auth: Supabase
- 배포: Vercel

## MCP
- Supabase: 백엔드 (DB, Auth, RLS)
- Vercel: 프론트엔드 배포

## API
- 원티드(Wanted) OpenAPI: 채용 정보 조회, 기업 정보, 직군/스킬 태그, 검색
  - 유료 AI 예측 엔드포인트(`/ai/pass`, `/ai/apply`)는 MVP에서 사용하지 않음 (계약 필요)
  - API 키는 절대 프론트엔드 코드/레포에 하드코딩 금지 — Vercel 환경변수 + 서버리스 함수 프록시로만 호출

## 파일 구조 및 참고 문서
- `PRD.md` : 전체 가치·방향성, 기능 명세
- `UI-UX-Guideline.md` : 프론트엔드 디자인 방향성 (색상, 타이포, 레이아웃, 톤앤매너)
- `Backend-Guideline.md` : 백엔드 구조 (DB 스키마, API 설계, 인증, 보안)

## 준수 사항
1. 코드를 작성할 때는 반드시 이 `CLAUDE.md`를 따르며, 프론트엔드 작업 시 `UI-UX-Guideline.md`, 백엔드 작업 시 `Backend-Guideline.md`를 함께 참고한다.
2. 코드 작성 후에는 반드시 테스트를 진행한다: (1) 프론트엔드 단독, (2) 백엔드 단독, (3) 프론트-백엔드 통합. 문제 발생 시 반드시 보고한다.
3. API 키, 비밀번호 등 민감정보는 절대 코드에 하드코딩하거나 커밋하지 않는다. `.env`는 반드시 `.gitignore`에 포함한다.
4. 코드 작성 서브에이전트의 작업이 끝나면 작업 종료를 보고하고, 곧바로 깃허브 push 서브에이전트가 작업을 이어간다.
5. 기능 단위(모듈)로 쪼개어 구현하고, 한 번에 여러 기능을 동시에 붙이지 않는다 (오류 추적 용이성 확보).
