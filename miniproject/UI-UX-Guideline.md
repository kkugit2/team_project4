# UI-UX-Guideline.md

프론트엔드 개발 시 참고하는 디자인 방향성 문서입니다. (이전 데모 아티팩트 `matchboard-dashboard.html`에서 사용한 톤을 기준으로 확장)

## 디자인 토큰

### 색상
| 이름 | HEX | 용도 |
|---|---|---|
| `--bg` | #F4F6F3 | 페이지 배경 |
| `--paper` | #FFFFFF | 카드/패널 배경 |
| `--ink` | #12241F | 기본 텍스트, 헤더 강조 |
| `--muted` | #63706A | 보조 텍스트, 설명문 |
| `--line` | #E1E5DF | 보더, 구분선 |
| `--teal` / `--teal-dark` | #0E7C66 / #0A5A4B | 매칭·강점·긍정 신호 (primary accent) |
| `--coral` | #D95F45 | 약점·경고·부족한 점 |
| `--amber` | #C08A2E | 랭킹·비교 기준선·하이라이트 |

### 타이포그래피
- 본문/UI: Pretendard (한글 가독성 우선)
- 데이터/수치(매칭 점수, %, 랭킹 숫자): JetBrains Mono — 숫자가 "측정된 값"이라는 인상을 주기 위해 별도 서체 사용
- 제목: Pretendard SemiBold 이상, 본문 대비 크기 차이를 명확히

### 시그니처 요소
- **매칭 게이지**: 원형 프로그레스(SVG arc) + 중앙 mono 폰트 숫자. 70% 이상 teal, 40~69% amber, 40% 미만 coral로 색상 자동 전환
- 이 게이지는 구직자용 기업 추천, 기업용 인재 랭킹, What-if 비교 등 매칭도가 나오는 모든 화면에서 동일하게 재사용 (일관성 확보)

## 레이아웃 원칙
- 최대 너비 1180px, 중앙 정렬
- 폼(입력)과 결과(출력)는 좌/우 2컬럼(360px : 1fr), 모바일에서는 1컬럼으로 전환
- 카드형 컴포넌트: `border-radius: 14px`, 얇은 보더(`1px solid var(--line)`), 그림자 대신 보더로 구분 (플랫한 톤 유지)

## 페이지별 가이드

### 랜딩페이지 (구직자용)
- 기업 카드형 그리드. 각 카드에 매칭 게이지 + 찜 아이콘 + 지원현황 뱃지
- 정렬/필터: 매칭도순, 최신순, 찜한 기업만 보기

### 랜딩페이지 (기업용)
- 지원자(익명화) 카드형 그리드. 랭킹 순 정렬, 인재상 일치도 게이지
- 스카웃 발송 버튼은 카드 내 보조 액션으로 배치 (주 액션인 "상세보기"보다 낮은 위계)

### 로그인/회원가입
- 가입 시 "구직자 / 기업" 탭 우선 선택 → 이후 폼 필드 분기
- 구직자: 아이디, 비밀번호, 학적, 학점, 자격증, 경력
- 기업: 아이디, 비밀번호, 회사명, 원하는 인재상(학점/기술스택/유사경험/인턴십 여부)
- 폼은 단계형(step) 대신 한 페이지 스크롤형으로 — MVP 단계에서 이탈률보다 구현 단순성 우선

### 마이페이지
- 좌측: 프로필 요약(학적/학점/자격증/경력), 우측: 자소서 분석 이력, 지원현황, 스카웃함(구직자) / 스카웃 발송 내역(기업)

### 기업 상세 페이지
- 상단: 회사 소개, 하단: 채용중 포지션 리스트(원티드 API 데이터)
- '지원하기' 버튼 클릭 시 새 탭으로 실제 기업 지원 페이지 이동 — 이탈 안내 문구 필요 ("외부 사이트로 이동합니다")

### 지원서 작성 후 피드백 페이지
- 자소서 원문 + 강점/약점 2단 카드(teal/coral 배경) + 경쟁자 비교 바차트
- 피드백은 "평가"가 아닌 "제안" 어조로 작성 (예: "~이 부족합니다" 대신 "~을 보완하면 좋습니다")

## 톤 & 라이팅
- 사용자가 통제하는 것을 기준으로 이름 짓기 (예: "매칭 재계산"이 아니라 "다시 확인하기")
- 실패/빈 상태는 원인과 다음 행동을 함께 제시 (예: "분석 결과가 없습니다 → 왼쪽에서 자소서를 입력해주세요")
- 존댓말 유지, 과도한 감탄사·이모지 지양
